Customer Feedback System

This project analyzes customer reviews collected from Google Maps to identify patterns in customer satisfaction and uncover opportunities for service improvement within a restaurant environment. The project was inspired by my experience working in a restaurant and my interest in understanding how customer feedback can be transformed into actionable business insights. The primary objective is to design and build an end-to-end analytics solution that collects, stores, and analyzes customer reviews to answer business questions related to customer experience, service quality, and operational performance. A Python-based ETL pipeline was developed to extract review data, clean and transform semi-structured information, and prepare it for storage in a relational database. The resulting dataset serves as the foundation for future SQL analysis and dashboard development.

Business Questions

This project seeks to answer the following business questions:

  • What are the most common themes in customer reviews
  • What factors are most frequently mentioned in negative reviews?
  • What are the most common themes in customer reviews
  • What factors are most frequently mentioned in positive reviews?
  • Are there any noticeable changes in customer sentiment over time?
  • Which aspects of the customer experience appear to have the greatest impact on customer satisfaction?
  • What actions could the restaurant team take to improve the customer experience and encourage repeat visits?

Data Source

The source data consists of publicly available customer reviews collected from Google Maps using the Apify Google Maps Reviews Scraper. The extracted reviews include information such as:

  • Customer display name
  • Review text
  • Overall review rating
  • Review date
  • Service context information (e.g., dine-in, takeout, meal type, price range)
  • Category ratings (Food, Service, Atmosphere)
  • Customer display name
  • Review text
Unlike traditional structured datasets, some review attributes are embedded within dictionary-like text fields. These fields require additional transformation before they can be analyzed effectively.

Tools & Technologies

  • Python: Developed an ETL pipeline to extract, clean, transform, and prepare customer review data for analysis.
  • Pandas: Cleaned raw review data, parsed semi-structured fields, handled missing values, standardized formats, and prepared the analysis-ready dataset.
  • Apify: Extracted Google Maps review data using a review scraping actor.
  • GitHub: Used for version control and project documentation.
  • Excel: Conducted initial exploratory data analysis to inspect raw review fields, missing values, and data structure.
  • Poetry: Managed project dependencies and maintained environment consistency.

ETL Pipeline

1. Extract

The extraction process uses the Apify Google Maps Reviews Scraper to collect customer reviews from Google Maps. The pipeline retrieves review information including customer details, review text, overall ratings, review dates, and contextual information describing the customer's visit experience. The extracted data is stored as a raw CSV file to preserve the original source data and provide traceability throughout the data pipeline.

2. Transform

The transform_data.py script converts the cleaned review dataset into an analysis-ready structure by parsing semi-structured fields, extracting relevant attributes, standardizing formats, and aligning columns with the target database schema.

One of the main challenges in this project was that some review attributes were stored as dictionary-like text fields. For example, visit information such as order type, meal time, and price range was embedded in the reviewContext column, while category-level ratings such as Food, Service, and Atmosphere were stored in the reviewDetailedRating column.

To transform these fields, I converted the dictionary-like strings into Python dictionaries using safe parsing logic. This allowed specific attributes to be extracted into separate columns such as order_type, meal_time, price_range_min, price_range_max, food_rating, service_rating, and atmosphere_rating.

The transformation process also standardized review dates by converting timestamp values into date fields, corrected encoding issues in price ranges, removed currency symbols, and split price ranges into minimum and maximum values. Open-ended price ranges such as 100+ were handled by preserving the minimum value while setting the maximum value as null to avoid making unsupported assumptions.

Finally, key columns were converted to appropriate data types. Review and customer identifiers were stored as strings to preserve their full values, while rating and price range fields were converted to nullable integer types to support missing values. Columns were then renamed to match the target schema, and intermediate raw parsing fields were removed from the final transformed dataset.

Current Status & Next Steps

This project is currently under active development. The data extraction, cleaning, and transformation pipeline has been completed, resulting in a structured dataset prepared for analysis and database storage. The next phase of the project focuses on designing and implementing a PostgreSQL database to store customer and review information using a relational schema. This stage will include creating tables, defining primary and foreign key relationships, and loading the transformed data into the database. Once the database layer is complete, the project will continue with SQL-based exploratory analysis to answer the business questions defined during the planning phase. The final stage will involve developing an interactive Tableau dashboard to visualize customer satisfaction trends, review patterns, and key drivers of positive and negative customer experiences. The dashboard will provide stakeholders with an intuitive way to explore customer feedback and identify opportunities for improvement.
